# Contributing to WandR Community

Thank you for helping make WandR better! This repository is the community hub for WandR — it's where we track bugs, discuss features, and maintain documentation.

## What We Accept

| Contribution Type | How |
|-------------------|-----|
| Bug reports | [Bug Report template](https://github.com/fabianwilliams/wandr-community/issues/new?template=bug_report.md) |
| Feature requests | [Feature Request template](https://github.com/fabianwilliams/wandr-community/issues/new?template=feature_request.md) |
| Questions | [Blank issue](https://github.com/fabianwilliams/wandr-community/issues/new) |
| Documentation improvements | Pull request |
| Privacy/Terms feedback | Issue or email |

## What We Don't Accept

- **Code contributions** — WandR's source code is proprietary. If you're a developer interested in the project or have found a security vulnerability, email support@adotob.com directly.
- **Design files or marketing materials** — Contact us via email instead.

---

## Reporting Bugs

Use the [Bug Report template](https://github.com/fabianwilliams/wandr-community/issues/new?template=bug_report.md) and include:

- **iPhone model and iOS version**
- **WandR app version** (found in App Store or app settings)
- **The trip query you typed** (if the bug is trip-related)
- **Steps to reproduce** — the more specific, the faster we can fix it
- **Screenshots** if applicable (but never include personal location data you're not comfortable sharing)

## Requesting Features

Use the [Feature Request template](https://github.com/fabianwilliams/wandr-community/issues/new?template=feature_request.md). The best requests explain:

- The problem you're trying to solve
- How you currently work around it
- Why other WandR users would benefit

## Asking Questions

Open a [blank issue](https://github.com/fabianwilliams/wandr-community/issues/new) with your question. No template needed.

---

## Issue Triage

Issues are reviewed and labeled by our team. Here's what the labels mean:

| Label | Meaning |
|-------|---------|
| `bug` | Confirmed bug |
| `enhancement` | Feature request |
| `question` | General question |
| `needs-info` | Waiting for more details from reporter |
| `backend-bug` | Bug in the backend/AI service |
| `ios-bug` | Bug in the iOS app |
| `ai-response` | Issue with AI-generated suggestions |
| `backlog` | Accepted for future consideration |
| `post-launch-review` | Will be evaluated after monitoring period |
| `priority-high` | Needs attention soon |
| `priority-low` | Nice to have |
| `duplicate` | Already reported |
| `wontfix` | Out of scope |

---

## Response Times

| Type | Target |
|------|--------|
| Bug reports | Acknowledged within 48 hours |
| Feature requests | Reviewed within 1 week |
| Questions | Answered within 72 hours |
| Security issues | Responded to within 24 hours (email support@adotob.com) |
| Pull requests (docs) | Reviewed within 1 week |

---

## Communication Guidelines

- Be respectful and constructive
- One issue per bug or feature request
- Search existing issues before creating a new one
- Include as much detail as possible — it helps us help you faster
- Don't include personal information (home address, exact GPS coordinates, etc.) in public issues

---

## Security

If you discover a security vulnerability, **do not open a public issue**. Email support@adotob.com directly. We take security seriously and will respond within 24 hours.
