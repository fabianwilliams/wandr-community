# Contributing to WandR Community

Thank you for helping make WandR better!

## How to Contribute

### Reporting Bugs
Use the [Bug Report template](https://github.com/fabianwilliams/wandr-community/issues/new?template=bug_report.md).

Include:
- iPhone model and iOS version
- WandR app version
- Steps to reproduce
- What you expected vs what happened
- Screenshots if applicable

### Requesting Features
Use the [Feature Request template](https://github.com/fabianwilliams/wandr-community/issues/new?template=feature_request.md).

The best feature requests explain:
- The problem you're trying to solve
- How you currently work around it
- Why you think other users would benefit

### Asking Questions
Open a [blank issue](https://github.com/fabianwilliams/wandr-community/issues/new) with your question.

---

## Issue Triage

Issues are labeled automatically:

| Label | Meaning |
|-------|---------|
| `bug` | Confirmed bug |
| `enhancement` | Feature request |
| `question` | General question |
| `needs-info` | Waiting on more details |
| `wont-fix` | Out of scope |
| `duplicate` | Already reported |

## Code Contributions

WandR's source code is not public. If you're a developer interested in contributing or have found a security issue, email support@adotob.com directly.

Be respectful. We're building something for adventurers — keep the spirit positive.
